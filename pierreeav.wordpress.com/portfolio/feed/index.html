Some Cisco switch has now a Linux environnement integrated to their OS : this is called the cisco IOx.
This allow to the switch to enable virtualization and to host small applications in the switch, with Dockers containers.

One of this application is the Guest Shell, which allow us to run a Linux environnement inside the switch!

With this Linux, we can create Python scripts to interact direcly with the switch with a special library, but we can also use the packet manager of Linux to install packets like tcpdumb, and monitor the traffic inside the switch, or install an apache Web server, or a FTP server inside the switch!

At the beginning, Cisco has created this functionality in order to facilitate the monitoring of switches from their central tool, but it also can be a lack of security.

]]> pierreeav Setting up a Honeypot server https://pierreeav.wordpress.com/portfolio/setting-up-a-honeypot-server/ Wed, 25 May 2022 07:29:26 +0000 http://pierreeav.wordpress.com/?post_type=jetpack-portfolio&p=115 Plus]]>

One of my biggest projects was the honeypot that I has created during my first internship.

First, what is a honeypot server?

It’s simply a false server, created and upgraded with intentional misconfigurations, to catch the hackers or analyse what they do once having infiltrate the server.

The goal of my Honeypot

My compagny, Flexnetwork, suffered, like almost all compagnies, from brute-force attacks against their publics servers, on VOIP and SSH services. Even if these attacks were innefective, they can slow down the network because a lot of packets can be send. So the CEO of Flex, Christopher LEAL, gave me a public server, with an OpenSSH service, and an Asterisk service (PBX), in order to analyse the IPs which attack these services and ban them.

Requirements

Firstly, I created a database, on MySQL, in which you will store all the attackers IPs.
You need to configure your SSH and Asterisk services, with very, very strong password.
You also need to install Fail2Ban, to prevent the constant stream of attacks.

Fail2Ban

Secondly, you need to configure Fail2ban, for both services, SSH and Asterisk.
Fail2Ban will automatically detect the connexion attempts on your server, and according to your configuration, ban the IPs addresses which spams a lot of failed connexion, directly with iptables, which allow you to configure the packet filter rules of the Linux kernel firewall.

Python script

On third part, I have created a python script, which retrieve ban IPs adresses from the Fail2Ban logs, with regex expressions, and send them to the MySQL database.

To run the script automatically, I just had a crontab, to run my python script everyday to fill the database.

RabbitMQ server and clients

To finish this project, the final objective was to spread all the blacklist IPs to the publics servers of Flexnetwork.

I realize this by creating two scripts in Golang programmation languages, which I can compute in binary file.

One first script to create a server, to retrieve all the IPs in the database and send them to the clients, and one second script, to create the client, which will receive the IPs and ban them on the machine from which it was launched.

So what we can do next, is to send the client code to all the public servers of the infrastructure, and via Ansible or via a common crontab, start all the clients at the same time, and start the server.
All the clients will receive the blacklist of IPs and ban them.
